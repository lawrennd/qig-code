# CIP-0007: Extend `MEPPSimulator` to Cover Stage 3 Plateau/Stalling Regime

## Summary

We propose to upgrade *MEPPSimulator* so it can simulate the *Stage 3* "long-plateau" period of Steepest-Entropy-Ascent (SEA) dynamics, including

1. adaptive *entropy-time* re-parametrisation,
2. detection of Fisher–metric sparsity (informational horizons), and
3. the attendant *stalling* / quasi-symmetry emergence.
   New diagnostics and visualisations will expose the slowing of external time $t$ while entropy time $\tau$ continues at unit speed.

## Tenet Alignment

This CIP extends the *Two-Stage Evolution: Dephasing to Isolation* tenet by implementing the critical Stage 3 plateau regime, where SEA flow freezes in sloppy directions and effective gauge/particle structures emerge. *Steepest Entropy Ascent as Dynamics* is central to the implementation, requiring adaptive entropy-time re-parametrisation to maintain dS/dτ = 1 while external time t slows dramatically. *Emergent Physics from Information Geometry* drives the plateau detection mechanism, using Fisher eigenvalue analysis to identify informational horizons and quasi-symmetry emergence. The simulation operates in *Natural Parameters as Canonical Coordinates*, with the adaptive RK45 integrator working in τ-space while tracking the relationship between entropy time and real time. *Mathematical Rigor and Systematic Investigation* ensures comprehensive testing and validation of the plateau physics implementation.

## Motivation

The current code halts after Stage 2 (isolation), missing the critical plateau where SEA flow freezes in sloppy directions and effective gauge/particle structures emerge. Without this regime we cannot:

* validate the *information-stalling* hypothesis,
* reproduce the predicted entropy-time dilation,
* compare with analytical plateau length estimates, or
* generate figures for Section 3 of the AISTATS paper.

## Detailed Description

### Key Concepts to Add

| Component                   | Role in Stage 3                                           | Implementation sketch                                                                                            |
| --------------------------- | --------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| *Entropy time $\tau$*     | Internal clock with $dS/d\tau=1$.                         | Integrator switches from fixed $dt$ to adaptive $d\tau$ with $dt = \|\nabla_\theta S\|_{G^{-1}}^{2}\,d\tau$.     |
| *Adaptive SEA step*       | Slows evolution when Fisher norm shrinks.                 | Replace `sea_evolution_step` with Runge–Kutta-Fehlberg on $\tau$.                                                |
| *Fisher spectrum monitor* | Detects sloppy directions / near horizons.                | Compute eigenvalues of $G(\theta)$ every few steps; flag plateau when $\min \lambda_i < \lambda_{\text{stall}}$. |
| *Plateau diagnostics*     | Visual proof of stalling.                                 | New arrays: `tau_history`, `lapse_history = dt/dτ`, `min_fisher_eig`.                                            |
| *Stage 3 loop*            | Continues until entropy within ε of $S_{\max}$ or τ\_max. | Add `simulate_stage3()` called after isolation.                                                                  |

### Code Modules Affected

* `sea_generator` → generalise to accept metric.
* `sea_evolution_step` → becomes `sea_step_tau` (adaptive).
* `simulate_evolution` → split into three stages; add `simulate_stage3`.
* Plotter → extra subplots for lapse function and Fisher eigenvalue trace.

## Implementation Plan

1. *Metric & Fisher utilities*

   * Add `compute_fisher(theta)` returning dense $G$ and its eigens.
   * Add `fisher_norm(theta)` → $\theta^{\!\top}G^{2}\theta$.

2. *Entropy-time integrator*

   * *Sub-task 1:* implement `sea_rhs_tau(theta)` returning $G^{-1}\nabla S$.
   * *Sub-task 2:* adaptive RK45 integrator on $\tau$; accumulate real-time $t$ via $dt = \lVert\nabla S\rVert^{2}_{G^{-1}} d\tau$.

3. *Plateau detection*

   * Define global `STALL_THRESHOLD = 1e-4`.
   * If `min_eig < threshold` for `window_len` steps → mark plateau start.

4. *Stage 3 loop & data logs*

   * Continue SEA steps until `(S_max - S) < ε` *or* `τ > τ_max`.
   * Record `tau_history`, `lapse_history`, `min_eig_history`.

5. *Visualisation upgrades*

   * New figure panel: `lapse(t)` (semilog) and Fisher min-eig trace.
   * Annotate plateau onset.

6. *Config flags & defaults*

   * Keep old behaviour when `enable_stage3=False`.
   * Default ε = 1e-3 nats, `τ_max=1e3`.

## Backward Compatibility

* All public APIs remain; new parameters are optional with sane defaults.
* Old scripts run unchanged (Stage 3 off).
* Pickled simulation data gains extra fields but earlier pickles still load.
